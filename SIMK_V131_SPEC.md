# Sim K v1.31 수정 스펙 — 계좌별 납입액 슬라이더 + ETF 배분 하위 구조

## 문제
현재 월 적립 모드에서 ISA 68% / 연금저축 20% / IRP 12% 고정 비율로 자동 배분됨.
사용자가 계좌별 납입액을 직접 조정할 수 없음.

## 수정 내용

### UI 구조 변경 (app/simk/page.tsx)

월 적립 / 연초 일시납 **둘 다** 아래 구조로 통일:

```
각 계좌 카드:
┌─────────────────────────────────────────────┐
│ ISA                           max 2,000만원/년 │
│ [슬라이더────────────●────] 150만원/월         │
│ 직접입력: [___150___] 만원/월                  │
│                                              │
│ ▼ ETF 배분 (접기/펼치기 토글)                  │
│   SCHD [슬라이더] 50%                         │
│   VOO  [슬라이더] 30%                         │
│   QQQ  [슬라이더] 20%                         │
│                        합계: 100% ✅          │
└─────────────────────────────────────────────┘
```

### 월 적립 모드 상세

기존: 총 월 납입액 슬라이더 하나 + 고정 비율 자동 배분
변경: 계좌별 개별 슬라이더 + 직접입력

```
ISA 납입액
  - 슬라이더: 0 ~ 166만원/월 (연 2,000만원 한도)
  - 직접입력 가능 (만원 단위, 쉼표 표시)
  - 기본값: 166만원/월

연금저축 납입액
  - 슬라이더: 0 ~ 125만원/월 (연 1,500만원 한도)
  - 직접입력 가능
  - 기본값: 50만원/월

IRP 납입액
  - 슬라이더: 0 ~ 25만원/월 (연 300만원 한도)
  - 직접입력 가능
  - 기본값: 25만원/월

총 월 납입액 표시 (읽기 전용, 자동 합산)
```

### 연초 일시납 모드 상세

기존과 동일하게 계좌별 연간 납입액 슬라이더 + 직접입력 유지
ETF 배분 하위 구조만 추가 (접기/펼치기)

### ETF 배분 하위 구조 (공통)

각 계좌 카드 하단에 접기/펼치기 토글로 ETF 배분 섹션 추가:
- "ETF 배분 ▼" 버튼 클릭 시 펼침/접힘
- 기본값: 펼쳐진 상태
- SCHD / VOO / QQQ 각 슬라이더 (0~100%)
- 합계 실시간 표시: 100% ✅ or 100% 아니면 ⚠️ 빨간색 경고
- 합계 != 100이면 결과 계산 안 됨 (경고 메시지)

슬라이더 입력 방식:
- 슬라이더로 대략 조정
- 옆에 % 숫자 직접 입력 가능
- 한 ETF 슬라이더 움직이면 나머지 자동 조정 (옵션, 구현 편의상 생략 가능)

### 타입 변경 (lib/simulatorK.ts)

```typescript
// 월 적립 모드
interface MonthlyAccount {
  monthlyWan: number  // 월 납입액 (만원)
  etfAlloc: { ticker: string; pct: number }[]
}

// 연초 일시납 모드
interface AnnualAccount {
  annualWan: number   // 연 납입액 (만원)
  etfAlloc: { ticker: string; pct: number }[]
}

interface SimKParams {
  mode: 'monthly' | 'annual'
  isa: MonthlyAccount | AnnualAccount
  pension: MonthlyAccount | AnnualAccount
  irp: MonthlyAccount | AnnualAccount
  // ...나머지 기존 파라미터 유지
}
```
